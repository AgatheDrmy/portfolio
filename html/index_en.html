Bot_Tommy

Waste-Eating Turtle


This project involves implementing a waste-collecting robot in a graphical interface using the turtle library. The turtle moves in a two-dimensional space. Its movement is done axis by axis (no diagonal movement). Additionally, its freedom of movement is limited (its movement cannot exceed a certain distance along each axis).

The simulation is divided into collection cycles. Each turn consists of several steps:

  • The turtle evaluates which pieces of waste it can collect based on the distance to them.
  • When a piece of waste is within reach, the turtle moves toward it and makes it disappear.
  • For each piece of waste, a new piece can be generated based on a fixed probability. The new piece will appear in an adjacent square to an original piece of waste.

The simulation will end if:

  • All the waste has been collected.
  • A maximum number of turns is reached.
  • All the waste is out of the turtleâ€™s reach.

Saint-Quentin

Study of the Tree Population


The goal of this project is the design and development of an application dedicated to the detailed study of tree heritage. It will allow the exploration and analysis of the tree heritage of the city of Saint-Quentin. The project provides a multidisciplinary approach, combining methods for data collection, processing, and visualization to provide valuable information about the city's trees. During this project, I mainly worked on displaying maps and the frontend of the web project.

Big Data

The Big Data part involves studying the data on the tree heritage of the city. To do this, we collected data from a CSV file obtained from the data.gouv website. We then cleaned the data (missing values, duplicates, other errors, etc.) to ensure its reliability. Afterward, we used R to display the data in various formats (charts, interactive maps, tables) using statistical models to analyze it.



IA

This part of the project aims to provide a detailed understanding of the different stages of a machine learning program, from data preparation to model evaluation. It also involves studying the evaluation of machine learning method performances by selecting the relevant metrics and correctly interpreting the results obtained. Various machine learning techniques are explored, including clustering for unsupervised learning and classification/regression for supervised learning. In this context, data on the trees are meticulously collected and modeled, focusing on predicting the age of the trees and creating an interactive map visualization to classify the trees by their size. Additionally, an artificial intelligence model is developed to predict the age of the trees using various features, and a warning system is implemented to anticipate trees likely to be uprooted during storms.



WEB

In the WEB section, the goal was the design and development of a tree heritage management application for the city of Saint-Quentin. There were several pages to create. Before developing the web application, we had to create the graphic charter and then design the site mockup using Figma. First, there was a homepage, followed by a form to add a tree to the database:

Finally, the last page allowed the visualization of trees on a map, with cluster predictions displayed on the map and the database shown in a table. There are also two buttons that lead to two different pages: one for displaying the prediction of tree ages and the other for uprooting trees.

Next, the development of the site (Front-end and Back-end) took place, along with the creation of the ER diagram (MCD) and the client-server interface. FlowTech

Creation of a Low-tech Solution



In this project, we had to design a Low Tech in a group of 6, starting from the invention to the construction of the final product. The goal was to create the construction and usage manual for this brand-new Low Tech.


What is a Low Tech?

A Low Tech is a simple, durable, and often local technology that addresses a specific need. Low Tech solutions prioritize minimal environmental impact. These technologies are robust, energy-efficient, and, above all, repairable. The materials used are mostly recycled materials.

Throughout this 3-month project, we used the agile method to organize ourselves. We learned to work together, reflected on the theory, and built this Low Tech with our own hands.
